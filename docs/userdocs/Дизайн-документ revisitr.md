## 2. Описание страниц

### 0. Лендинг

**Реф**: [teyca.ru](https://teyca.ru/)

### 1. Главная страница (дашборд)

Виджеты с основной информацией с остальных страниц.

**Фильтры**: период (сегодня / вчера / эта неделя / последние 7 дней / этот месяц / последние 30 дней / этот год / последний год / произвольный) + сегмент аудитории + бот + заведение.

**Виджеты**:
- Выручка участников программы за период
- Средний чек (участники vs. неучастники)
- Количество новых регистраций в боте
- Активные пользователи бота
- Количество отправленных рассылок и их открываемость

---

### 2. Аналитика

Страница с виджетами — все цифры и статистика.

**Фильтры**: те же, что на дашборде (период + сегмент + бот + заведение).

#### 2.1. Продажи

Бизнес-метрики по продажам.

**Виджеты** (общее + лояльность):
- Количество транзакций
- Количество клиентов
- Средний чек
- Сумма чеков
- Частота покупок

\+ графики по каждому пункту

#### 2.2. Лояльность

Данные по системе лояльности.

**Виджеты**:
- Новых пользователей
- Активных пользователей
- Бонусов накоплено
- Бонусов списано

\+ отдельный сегмент с действиями в боте (куда нажимали, воронки и т.д.)

**Аудитория** (круговые диаграммы):
- Пол
- Возраст
- Процент лояльности (без / Telegram / Wallet)
- ОС

#### 2.3. Рассылки

Данные по всем рассылкам.

**Виджеты**:
- Отправлено
- Открыто
- Конверсий в (бронирование / заказы / действия в боте)

---

### 3. Клиенты

Данные по всем клиентам, зарегистрированным в системе лояльности.

#### 3.1. Клиенты

Сверху виджеты со статистикой, снизу — таблица всех клиентов + фильтры. При клике открывается профиль клиента.

**Виджеты**:
- Общее количество участников программы лояльности
- Общее количество накопленных баллов

**Таблица**:

| Карта | Имя | Уровень | Бот / Заведения | Сегмент (теги) | Сумма покупок | Кол-во покупок | Баланс баллов |
|-------|-----|---------|-----------------|----------------|---------------|----------------|---------------|

**Профиль клиента**:
- Имя, пол, возраст
- Дата регистрации
- Текущий уровень
- Сегмент (теги)
- Активированные боты
- Сумма покупок, количество покупок
- Текущая сумма баллов, потрачено баллов
- Устройство (ОС)
- История заказов

#### 3.2. Сегменты

Страница с созданием сегментов (фильтров/тегов).

##### 3.2.1. RFM

Авто-тегирование по параметрам: давность, частота, деньги.

Круговая диаграмма с распределением по категориям + таблица:

| Сегмент | Кол-во клиентов | % клиентов | Кол-во покупок | % покупок | Среднее кол-во покупок | Сумма покупок | Средний чек |
|---------|-----------------|------------|----------------|-----------|------------------------|---------------|-------------|

##### 3.2.2. Мои сегменты

Создание своих сегментов по любым доступным параметрам:

- Пол, возраст
- Дата регистрации
- Уровень в программе
- Сегмент, заведения
- Сумма покупок (за любой период)
- Количество покупок (за любой период)
- Количество посещений (за любой период)
- Среднее количество посещений (за любой период)
- Текущий баланс
- Потрачено всего (за любой период)
- Накоплено всего (за любой период)
- Устройство
- Telegram / Wallet

> Если клиент попадает в сегмент из 3.2.1 или 3.2.2, ему в профиле присваивается соответствующий тег.

### 4. Лояльность

Создание и настройка программы лояльности.

> У одного бота может быть только одна программа лояльности, но у одной программы лояльности может быть несколько ботов.

**Доступные варианты (сейчас)**:
- **Бонусная** (многоуровневая) — накапливаются и списываются бонусы, % растёт от общей суммы покупок за период
- **Дисконтная** — % скидки в зависимости от общей суммы покупок за период

**На будущее**:
- **Геймификация** — при накоплении определённой суммы заказов за период получаешь N (скидка общая / скидка на заказ / конкретный товар/услуга и т.д.)

#### 4.1. Мои программы

Список всех программ (карточками или списком) и их настройка.

**Создание**:
- Название
- Формат

**Настройки** — таблица с тремя столбцами, пустая строка + кнопка «Добавить ещё»:

Если **бонусная**:

| Название уровня | Порог суммы (от) | % начисляемых бонусов |
|-----------------|------------------|-----------------------|

Если **дисконтная**:

| Название уровня | Порог суммы (от) | % скидки |
|-----------------|------------------|----------|

#### 4.2. Wallet*

Страница создания и настройки Wallet.

### 5. Рассылки

Создание и аналитика рассылок.

#### 5.1. Запустить рассылку

Рассылка по заранее созданным сегментам (раздел «Клиенты») или произвольным фильтрам. Можно создать на сайте (ограниченный функционал) или перейти в Telegram.

**Поля на сайте**:
- Для какой аудитории
- Когда
- Текст сообщения
- Прикреплённые файлы

> **TODO**: продумать, как организовать создание рассылки через Telegram.

#### 5.2. Авто-рассылки

Автоматические рассылки по триггерам. Два шага: создание сценария + создание сообщения.

**Сценарии**:
- Не был N времени
- Пришёл в N-й раз
- Накопил N бонусов
- Повысил уровень до X
- Купил позицию X
- День рождения

\+ готовые сценарии по RFM

**Поля на сайте**:
- Для какой аудитории
- Когда
- Текст сообщения
- Прикреплённые файлы

#### 5.3. Архив рассылок

Список архивных рассылок и статистика по ним.

### 6. Акции и промокоды

Создание акций и промокодов.

**Реф**: [YouTube](https://youtu.be/qnx9gmpQVzY?si=N0AqLnE4yOG2zuzz&t=1144)

#### 6.1. Создать акцию

Конструктор акций в несколько шагов.

**Шаг 1 — Создание**:
- Название
- Период действия
- Можно использовать неограниченное количество раз? (да/нет; если нет — поле для числа)
- Сочетается с другими акциями? (да/нет)
- Сочетается с программой лояльности? (да/нет)

**Шаг 2 — Условия**:

*Фильтр по клиентам* (+ показывает % подходящих):
- ID, имя, пол, возраст / дата рождения, город
- Номер подтверждён
- Дата регистрации, уровень в программе, сегмент
- Активированные боты, заведения
- Сумма покупок (за любой период)
- Количество покупок (за любой период)
- Количество посещений (за любой период)
- Среднее количество посещений (за любой период)
- Текущий баланс
- Потрачено всего (за любой период)
- Накоплено всего (за любой период)
- Устройство, Telegram / Wallet
- Дата последней покупки

*Условия покупки*:
- Факт покупки
- Покупка X
- Количество позиций в чеке
- Сумма чека
- Дни недели / даты

*Триггер на событие*:
- День рождения
- Регистрация
- Активация бота / Wallet
- Последняя покупка

**Шаг 3 — Результат**:
- Скидка
- Бонусы
- Обновление данных (присвоение тега или изменение уровня)
- Рассылка

#### 6.2. Мои акции

Список всех активных акций.

#### 6.3. Создать промокод

**Поля**:
- Название (для внутреннего пользования)
- Период действия
- Сам промокод (например: NEW10, OLD25) — опционально
- Размер скидки (%)

**Условия**: if/else с привязкой к фильтрам (фильтр по клиентам, триггер на событие, триггер на дату, условия покупки) — аналогично акциям.

#### 6.4. Мои промокоды

Список всех активных промокодов.

#### 6.5. Архив

Список всех архивных акций и промокодов.

### 7. Мои боты

Создание и настройка Telegram-ботов.

#### 7.1. Создать бота

Поп-ап или отдельная страница. Ссылка на гайд по созданию бота и получению API-ключа.

**Поля**:
- Название
- Ссылка на бота
- API-ключ
- Точки продаж*

\+ кнопка «Продолжить»

> *Выпадающее меню с галочками для привязки точек продаж (из раздела «Точки продаж»). Если точек ещё нет — кнопка «Создать» с переходом на страницу создания и возвратом обратно с сохранением данных.*

#### 7.2. Мои боты

Карточки:
- Название + аватарка
- Ссылка
- Статус
- Кнопка «Настроить»

#### 7.3. Настройка бота

**Настройки**:
- Подключение и настройка модулей*
- Подключённые точки продаж
- Приветственное сообщение
- Доступные кнопки (конструктор бота, в идеале с live-превью)
- Форма регистрации

---

**\* Модули**:
- Программа лояльности
- Бронирование
- Меню
- Обратная связь / поддержка
- *(позже)* Ланч, N-ое X бесплатно

**Настройка модулей**:

| Модуль | Настройки |
|--------|-----------|
| **Бронирование** | Форма: дата, время, кол-во человек, номер для связи*, комментарий. Уведомления: номер / Telegram / WhatsApp / Max |
| **Меню** | Загрузить PDF или картинки + предпросмотр |
| **Обратная связь** | Кнопка «Связаться с рестораном» (название кастомизируется). Куда приходит: номер / Telegram / WhatsApp / Max |

> *Номер для связи: кнопка «Отправить номер» из Telegram (если не отправлял ранее) / кнопка «Мой» (если отправлял ранее) / кнопка «Ввести номер».*

**Точки продаж**: к одному боту можно привязать несколько точек. В настройках «Доступных кнопок» можно включить выбор точки продаж, если в разных заведениях разное меню, график, контакты и т.д.

**Форма регистрации** — настройка полей при первом `/start` (можно отключить):
- Имя
- Пол
- Город
- Возраст / дата рождения
- **Отправить номер** (кнопка в Telegram)

**Нужны сейчас**:
- iiko
- r_keeper
- 1C

**Цели**:
1. Связать профиль клиента из раздела «Клиенты» с данными по его заказам из этих систем
2. Получить данные по всем продажам (не из системы лояльности) для аналитики

### 10. Тарифы

Выбор тарифа и переход в корзину.

| Тариф | Возможности |
|-------|-------------|
| **Базовый** | Только программа лояльности, 1 бот максимум |
| **PRO** | Программа лояльности + все стандартные модули, 3 бота максимум |
| **Enterprise** | Всё выше + личный менеджер + оплата с юр. лица + кастомные модули, неограниченное кол-во ботов |
