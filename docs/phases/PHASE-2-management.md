## Задачи

### 2.1 Клиенты

#### Backend
- [ ] Расширение `bot_clients`: пол, возраст, дата рождения, город, ОС, сегменты (tags JSONB)
- [ ] Endpoint: `GET /clients` — с пагинацией, сортировкой, фильтрами
- [ ] Endpoint: `GET /clients/:id` — полный профиль + история заказов
- [ ] Endpoint: `PATCH /clients/:id` — редактирование тегов/сегментов
- [ ] Endpoint: `GET /clients/stats` — виджеты (общее кол-во, общий баланс)

#### Frontend
- [ ] Таблица клиентов (TanStack Table):
  - Колонки: карта, имя, уровень, бот/заведение, сегмент, сумма покупок, кол-во, баланс
  - Серверная пагинация + фильтры
  - Поиск по имени/телефону
- [ ] Профиль клиента (модал или отдельная страница):
  - Основная информация
  - Текущий уровень и баланс
  - История заказов (таблица)
  - Теги/сегменты
- [ ] Виджеты статистики сверху

### 2.2 Дашборд

#### Backend
- [ ] Endpoint: `GET /dashboard/widgets` — агрегированные данные
  - Выручка участников за период
  - Средний чек (участники vs неучастники)
  - Новые регистрации
  - Активные пользователи
  - Статистика рассылок
- [ ] Фильтры: период, сегмент, бот, заведение

#### Frontend
- [ ] Виджеты-карточки с числами + тренд (↑↓)
- [ ] Фильтры сверху (период, сегмент, бот, заведение)
- [ ] Графики (recharts или chart.js):
  - Выручка по дням
  - Новые регистрации по дням
  - Средний чек

### 2.3 Рассылки

#### Backend
- [ ] Миграция: `campaigns` (id, org_id, type[manual|auto], status, audience_filter JSONB, message, scheduled_at, sent_at, stats JSONB)
- [ ] Миграция: `campaign_messages` (id, campaign_id, client_id, status[pending|sent|opened|failed], sent_at)
- [ ] Entity: `Campaign`, `CampaignMessage`
- [ ] Usecase: `campaigns`
  - Создание ручной рассылки (фильтр аудитории + текст + файлы)
  - Подсчёт охвата (сколько клиентов попадают в фильтр)
  - Отправка через Telegram-бота (очередь в Redis)
  - Отслеживание статусов
- [ ] Авто-сценарии:
  - Не был N времени
  - Пришёл в N-й раз
  - Накопил N бонусов
  - Повысил уровень
  - День рождения
- [ ] Controller: CRUD + `POST /campaigns/:id/send`
- [ ] Bot integration: отправка сообщений клиентам

#### Frontend
- [ ] Создание рассылки:
  - Выбор аудитории (фильтры + отображение % подходящих)
  - Время отправки (сейчас / запланировать)
  - Текст + файлы
  - Предпросмотр
- [ ] Авто-сценарии:
  - Список шаблонов-триггеров
  - Настройка условий + сообщения
- [ ] Архив рассылок:
  - Таблица: дата, тип, охват, статус, открытия
  - Детали рассылки
