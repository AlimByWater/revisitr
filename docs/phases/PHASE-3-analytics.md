# Фаза 3: Аналитика и интеграции

> Цель: глубокая аналитика, сегментация, интеграции с POS-системами, промокоды.

## Статус: ⏳ Не начата

## Зависимости: Фаза 2 завершена

---

## Задачи

### 3.1 Аналитика

#### Backend
- [ ] Endpoint: `GET /analytics/sales` — метрики продаж
  - Количество транзакций
  - Количество клиентов
  - Средний чек
  - Сумма чеков
  - Частота покупок
  - + графики по каждому показателю (данные за период)
- [ ] Endpoint: `GET /analytics/loyalty` — метрики лояльности
  - Новые пользователи
  - Активные пользователи
  - Бонусов накоплено / списано
  - Аудитория: пол, возраст, ОС, % лояльности (круговые)
  - Действия в боте (воронки)
- [ ] Endpoint: `GET /analytics/campaigns` — метрики рассылок
  - Отправлено / открыто
  - Конверсии
- [ ] Фильтры: период + сегмент + бот + заведение (единые для всех)
- [ ] Оптимизация: материализованные представления или pre-computed aggregates для тяжёлых запросов

#### Frontend
- [ ] Страница «Продажи»:
  - Виджеты с числами
  - Линейные графики по каждому показателю
  - Сравнение: участники vs неучастники
- [ ] Страница «Лояльность»:
  - Виджеты (новые, активные, бонусы)
  - Круговые диаграммы (пол, возраст, ОС, % лояльности)
  - Воронки действий в боте
- [ ] Страница «Рассылки»:
  - Виджеты (отправлено, открыто, конверсии)
  - Таблица по рассылкам

### 3.2 Сегментация

#### Backend
- [ ] Миграция: `segments` (id, org_id, name, type[rfm|custom], filter JSONB, auto_assign bool)
- [ ] RFM-анализ:
  - Автоматический расчёт Recency/Frequency/Monetary для каждого клиента
  - Классификация по категориям (чемпионы, лояльные, спящие, потерянные...)
  - Cron job для пересчёта (scheduler)
- [ ] Кастомные сегменты:
  - Фильтр по любым параметрам (пол, возраст, сумма покупок, кол-во визитов и т.д.)
  - Автоприсвоение тегов при попадании в сегмент
- [ ] Controller: CRUD сегментов + `GET /segments/:id/clients`

#### Frontend
- [ ] RFM-страница:
  - Круговая диаграмма с распределением по категориям
  - Таблица: сегмент, кол-во клиентов, %, покупки, средний чек
- [ ] Кастомные сегменты:
  - Конструктор фильтров (выпадающие поля + условия)
  - Предпросмотр кол-ва попадающих клиентов

### 3.3 Интеграции с POS

#### Backend
- [ ] Миграция: `integrations` (id, org_id, type[iiko|rkeeper|1c], config JSONB, status, last_sync_at)
- [ ] Миграция: `external_orders` (id, integration_id, external_id, client_id, items JSONB, total, created_at)
- [ ] iiko integration:
  - API клиент для iiko Cloud
  - Синхронизация заказов
  - Привязка клиентов по телефону/ID
- [ ] r_keeper integration:
  - API клиент
  - Аналогичная логика
- [ ] 1C integration:
  - Webhook receiver или polling
- [ ] Scheduler: периодическая синхронизация заказов

#### Frontend
- [ ] Страница «Интеграции»:
  - Карточки доступных интеграций
  - Настройка подключения (API key, URL и т.д.)
  - Статус синхронизации
  - Логи последних синхронизаций

### 3.4 Акции и промокоды

#### Backend
- [ ] Миграция: `promotions` (id, org_id, name, type[discount|bonus|tag_update|campaign], conditions JSONB, result JSONB, period, usage_limit, active)
- [ ] Миграция: `promo_codes` (id, org_id, code, discount_percent, period, conditions JSONB, usage_count, active)
- [ ] Миграция: `promotion_usages` (id, promotion_id, client_id, used_at)
- [ ] Usecase: конструктор акций (3 шага: создание → условия → результат)
- [ ] Controller: CRUD для акций и промокодов
- [ ] Bot integration: применение акций/промокодов при транзакциях

#### Frontend
- [ ] Конструктор акций (3 шага из дизайн-документа):
  1. Создание (название, период, лимиты, сочетаемость)
  2. Условия (фильтр клиентов + условия покупки + триггеры)
  3. Результат (скидка / бонусы / обновление данных / рассылка)
- [ ] Список акций + архив
- [ ] Промокоды: создание, список, архив

## Критерии завершения

- [ ] Аналитика: все виджеты и графики из дизайн-документа работают
- [ ] RFM-сегментация: автоматический расчёт и визуализация
- [ ] Кастомные сегменты: конструктор фильтров, автоприсвоение тегов
- [ ] Хотя бы 1 POS-интеграция полностью рабочая (iiko)
- [ ] Конструктор акций: полный 3-шаговый процесс
- [ ] Промокоды: создание и применение через бота
